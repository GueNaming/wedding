# 💍 모바일 청첩장 운영 배포 가이드

본 가이드는 로컬 개발 환경에서 완성된 청첩장을 실제 운영 도메인에 배포하고, 네이버 지도 및 티맵 등 주요 기능을 검증하는 단계를 설명합니다.

---

## 1단계: 도메인 구매 및 호스팅 설정

### 1. 도메인 구매
- **추천 사이트**: 가비아(Gabia), 후이즈(Whois), AWS Route 53 등
- **팁**: `규남은주결혼.com` 또는 `gyunam-eunju-wedding.com`과 같이 기억하기 쉬운 이름을 추천합니다.

### 2. 무료 호스팅 배포 (Vercel 또는 Netlify 추천)
본 프로젝트는 Vite 기반이므로 Vercel이 가장 호환성이 좋고 빠릅니다.
1. [Vercel](https://vercel.com/) 사이트 가입 (GitHub 연동 권장)
2. `New Project` -> GitHub 저장소 선택
3. Build Command: `npm run build`
4. Output Directory: `dist`
5. `Deploy` 클릭 시 바로 임시 URL(`...vercel.app`)로 배포됩니다.
6. 구매한 도메인을 Vercel 설정(`Settings -> Domains`)에서 연결하세요.

---

## 2단계: 네이버 지도 API 설정 (운영 도메인 등록)

로컬에서 지도가 안 뜨는 이유는 네이버 클라우드 플랫폼에 **운영 도메인이 등록되지 않았기 때문**입니다.

1. [네이버 클라우드 플랫폼 콘솔](https://console.ncloud.com/) 접속
2. `Services` -> `AI·NAVER API` -> `Maps` 선택
3. 등록된 Application 선택 후 `변경` 클릭
4. **서비스 환경 설정** -> **Web 서비스 URL**에 아래 두 항목을 추가:
   - `http://localhost:5173` (로컬 테스트용)
   - `https://여러분의-운영-도메인.com` (실제 운영용)
5. `Client ID`를 복사하여 `index.html`의 다음 부분에 적용:
   ```html
   <script src="https://oapi.map.naver.com/openapi/v3/maps.js?ncpKeyId=여러분의_ID"></script>
   ```

## 3단계: 운영 환경 기능 테스트 가이드

배포 후 실제 모바일 기기(iPhone/Android)에서 다음 항목을 테스트하세요.

### 1. 티맵(T-Map) 및 지도 앱 연동
- **테스트 방법**: 모바일 브라우저로 접속하여 '티맵' 버튼 클릭.
- **예상 동작**: 티맵 앱이 설치되어 있다면 자동으로 경로 안내 앱이 실행됩니다.
- **참고**: 로컬(PC) 환경에서는 앱 호출(`tmap://`)이 지원되지 않아 흰 화면이 뜨거나 반응이 없을 수 있습니다. 운영 도메인에서는 정상 작동합니다.

### 2. 클립보드 복사 (계좌번호)
- **테스트 방법**: '복사' 버튼 클릭.
- **예상 동작**: "복사되었습니다!" 알림이 뜨고 실제로 붙여넣기가 가능한지 확인.
- **참고**: 최신 브라우저의 보안 정책상 **HTTPS** 환경에서만 클립보드 복사 API가 정상 작동합니다. Vercel 배포 시 자동 적용되는 HTTPS를 통해 확인하세요.
